# YouTube 메타데이터 자동 생성기

이 스킬은 완성된 강의 자료를 분석하여 YouTube 업로드에 최적화된 메타데이터를 자동으로 생성합니다.

| 생성 항목 | 설명 |
|---|---|
| **제목 후보 3개** | 60자 이내, SEO 최적화, A/B 테스트용 |
| **설명 (Description)** | 훅 → 타임스탬프 → 학습 내용 → 링크 → 해시태그 |
| **태그** | 500자 이내, 한국어+영어 혼합 |
| **썸네일 문구** | 15자 이내 메인 문구 + 서브 문구 |
| **썸네일 컨셉** | 배경색, 레이아웃, 아이콘 디자인 명세 |
| **재생목록 제안** | 적합한 재생목록 카테고리 |
| **카드/최종 화면** | 연관 영상/구독 CTA 제안 |

## 워크플로우

### 1단계: 입력 수집

사용자로부터 강의 식별 정보를 수집합니다.

**입력 형식 (다음 중 하나):**
- 강의 번호: `0001`
- 폴더 경로: `lectures/0001-터미널-생산성-설정/`
- 현재 작업 중인 강의라면: "현재 강의" 또는 폴더 자동 감지

**입력이 모호한 경우:**
- `lectures/` 디렉토리를 확인하여 폴더 목록을 제시
- 사용자가 선택할 수 있도록 안내

### 2단계: 강의 자료 분석

강의 폴더에서 두 파일을 읽고 핵심 정보를 추출합니다.

**읽을 파일:**
```
lectures/[폴더명]/lecture-script.md
lectures/[폴더명]/README.md
```

**추출할 정보:**

_lecture-script.md에서:_
- 강의 제목 및 핵심 주제
- 각 섹션 제목과 예상 시간 → 타임스탬프 생성
- 수치 데이터 (예: "82% 단축", "5배 향상") → 제목/훅 소재
- 핵심 포인트 (`[🎯 핵심 포인트]` 구간) → SEO 키워드
- 다루는 도구/기술 목록 → 태그 소재
- 인트로 첫 단락 → 훅 문구 소재

_README.md에서:_
- 빠른 시작 체크리스트 항목 → 학습 내용 요약
- 도구명과 링크 → 설명 섹션 링크 목록
- 핵심 명령어/단축키 → 태그 보완

### 3단계: 메타데이터 생성

추출한 정보를 바탕으로 `references/youtube-metadata-template.md`의 형식에 따라 전체 메타데이터를 작성합니다.

**제목 3개 작성 원칙:**
- 모두 60자 이내 (초과 시 재작성)
- 각각 다른 각도로 접근:
  1. **수치/결과 중심**: "연간 44시간 절약하는 터미널 설정법 | 개발자 생산성"
  2. **문제/해결 중심**: "맥 개발자 터미널, 이렇게 세팅하면 바로 달라집니다 | Oh My Zsh"
  3. **대상/도구 중심**: "macOS 터미널 완전정복 | iTerm2 + Powerlevel10k + fzf"
- 핵심 키워드를 앞쪽에 배치
- 과장 없이 실제 강의 내용 반영

**설명 작성 원칙:**
- 훅(2~3줄): 강의의 가장 강력한 수치/가치 제시, 시청자가 얻을 것을 명확히
- 빈 줄
- 타임스탬프: `00:00 제목` 형식, 인트로부터 클로징까지 모든 섹션
- 빈 줄
- 학습 내용: `✅ 항목` 형식으로 3~5개
- 빈 줄
- 강의에서 사용한 도구 링크: README.md의 "유용한 링크" 섹션 참조
- 빈 줄
- 강의 자료 GitHub 링크: 아래 규칙으로 자동 생성
  ```
  https://github.com/minsuhya/gillilab-lecture-factory/tree/master/lectures/[폴더명]/
  ```
  예: `lectures/0001-터미널-생산성-설정/` → `https://github.com/minsuhya/gillilab-lecture-factory/tree/master/lectures/0001-%ED%84%B0%EB%AF%B8%EB%84%90-%EC%83%9D%EC%82%B0%EC%84%B1-%EC%84%A4%EC%A0%95/`
  단, URL 인코딩 없이 폴더명 그대로 사용해도 GitHub이 처리하므로 **원본 폴더명을 그대로 사용**한다:
  `https://github.com/minsuhya/gillilab-lecture-factory/tree/master/lectures/0001-터미널-생산성-설정/`
- 빈 줄
- 채널 링크 (고정값, 매 강의 동일하게 삽입):
  ```
  📺 길리랩 채널
  • Tech Log: https://techlog.gillilab.com
  • Tistory: https://rupijun.tistory.com
  • Blogger: https://gillilab.blogspot.com
  • Instagram: https://www.instagram.com/gillilab/
  • Threads: https://www.threads.com/@gillilab
  ```
- 빈 줄
- 해시태그: `#태그1 #태그2 ...` (설명 맨 끝)

**태그 작성 원칙:**
- 총 500자 이내
- 한국어 태그와 영어 태그 혼합
- 넓은 범위 → 좁은 범위 순서 배치
  - 넓은: `개발`, `프로그래밍`, `개발자`
  - 중간: `터미널`, `맥`, `macOS`
  - 좁은: `iTerm2`, `OhMyZsh`, `Powerlevel10k`
- 쉼표로 구분

**썸네일 문구 원칙:**
- 메인 문구: 15자 이내, 강렬한 숫자나 동사 사용
  - 예: "연간 44시간 절약" / "터미널 완전정복"
- 서브 문구: 10자 이내, 도구명이나 보조 설명
  - 예: "Oh My Zsh" / "맥 개발자 필수"

**썸네일 컨셉 원칙:**
- 배경색: 기술 특성에 맞는 색상 (터미널→어두운 배경, 웹→밝은 배경)
- 레이아웃: 텍스트 + 아이콘/로고 배치 명세
- 강의 내용을 시각적으로 표현할 아이콘/이미지 제안

### 4단계: 파일 저장

생성한 메타데이터를 파일로 저장합니다.

**저장 경로:** `lectures/[번호]-[슬러그]/youtube-metadata.md`

예: `lectures/0001-터미널-생산성-설정/youtube-metadata.md`

파일 형식은 `references/youtube-metadata-template.md`를 따릅니다.

### 5단계: 품질 검증

저장 전 다음 항목을 확인합니다.

**제목 검증:**
- [ ] 제목 3개 모두 60자 이내인가?
- [ ] 각 제목이 서로 다른 각도로 작성되었는가?
- [ ] 핵심 키워드가 제목 앞쪽에 위치하는가?

**설명 검증:**
- [ ] 훅이 첫 2~3줄 이내인가?
- [ ] 타임스탬프가 `00:00` 형식인가?
- [ ] 모든 섹션이 타임스탬프에 포함되었는가?
- [ ] 해시태그가 맨 끝에 위치하는가?

**태그 검증:**
- [ ] 태그 전체 합산이 500자 이내인가?
- [ ] 한국어와 영어 태그가 혼합되었는가?

**썸네일 검증:**
- [ ] 메인 문구가 15자 이내인가?
- [ ] 서브 문구가 10자 이내인가?

### 6단계: 사용자 검토 및 개선

파일 생성 완료 후 사용자에게 다음을 안내합니다:

1. 생성된 파일 경로 안내: `lectures/[폴더]/youtube-metadata.md`
2. 제목 3개 미리보기 (글자 수 포함)
3. 수정 요청 시 Edit 도구로 반영

## 타임스탬프 생성 가이드

`lecture-script.md`의 `[약 N분]` 표시를 기반으로 누적 시간을 계산합니다.

```
[인트로] 약 5분          → 00:00
[섹션 1] 약 5분          → 00:05
[섹션 2] 약 10분         → 00:10
[섹션 3] 약 5분          → 00:20
...
```

시간이 명시되지 않은 경우, 섹션 분량(줄 수)으로 추정합니다.

형식: `MM:SS 섹션 제목` (1시간 미만), `H:MM:SS 섹션 제목` (1시간 이상)
