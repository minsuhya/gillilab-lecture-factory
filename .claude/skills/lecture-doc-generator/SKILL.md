# 강의 자료 자동 생성기

이 스킬은 기술 주제에 대한 실무 중심 한국어 강의 자료를 자동으로 생성합니다.
공식 문서를 참고하여 **두 개의 파일**을 생성합니다:

| 파일 | 대상 | 형식 |
|---|---|---|
| `lecture-script.md` | 강의자 | 격식체 낭독 스크립트 + 큐 마커 |
| `README.md` | 시청자 | 명령어/참조표 중심 실용 가이드 |

## 워크플로우

### 1단계: 기본 정보 수집

사용자로부터 다음 정보를 수집합니다:

**필수 정보:**
- **주제어**: 강의할 기술/도구/라이브러리 이름 (예: "FastAPI", "Docker", "React Hooks")
- **간단한 설명**: 해당 기술이 무엇인지 한 줄로 (예: "Python 웹 프레임워크", "컨테이너 플랫폼")

**선택 정보:**
- **공식 문서 URL**: 있다면 제공 (없으면 자동 검색)
- **강조할 내용**: 특히 다루고 싶은 기능이나 주제
- **대상 청중**: 초급/중급/고급 개발자
- **실무 시나리오**: 실제 프로젝트에서 어떻게 사용하는지

**정보 수집 방법:**

사용자가 충분한 정보를 제공한 경우:
- 바로 2단계로 진행

사용자가 간단하게 요청한 경우 (예: "FastAPI 강의 자료 만들어줘"):
- 주제어를 파악하고 나머지는 자동으로 처리
- 공식 문서 자동 검색
- 표준 구조로 문서 생성

### 2단계: 공식 문서 조사

**WebSearch 사용:**
```
"{주제어} official documentation"
"{주제어} getting started guide"
"{주제어} tutorial"
```

**WebFetch 사용:**
제공된 URL 또는 검색된 공식 문서 URL에서 다음 정보 추출:
- 핵심 개념 및 용어
- 설치 방법 및 요구사항
- 기본 사용법 및 API
- 베스트 프랙티스
- 실전 예제

**중요 원칙:**
- 공식 문서의 정확성을 최우선으로
- 최신 버전 정보 확인
- 한국어로 의역하되 기술 용어는 원문 유지

### 3단계: 콘텐츠 구조 설계

수집한 정보를 바탕으로 강의 전체 구조를 설계합니다.

**기본 섹션 구성:**

1. 개요 (왜 필요한가, 시간/성능 수치)
2. 시작하기 (사전 요구사항, 설치, 초기 설정)
3. 핵심 개념 (용어, 아키텍처, 레이어 구분)
4. 필수 도구 설치 및 설정
5. 주요 기능 / 플러그인
6. 현대적 대안 도구 (해당 시)
7. 실무 예제 3가지
8. 베스트 프랙티스
9. 트러블슈팅
10. 추가 학습 자료
11. 결론

**구조 조정 원칙:**
- 주제에 따라 섹션 추가/제거/재배치
- 프로그래밍 라이브러리: API 레퍼런스 추가
- DevOps 도구: 아키텍처 및 배포 전략 강화
- 개발 도구: 단축키 및 워크플로우 최적화 추가

### 4단계: 콘텐츠 작성

각 섹션에 대해 다음 원칙을 따릅니다:

**개요 섹션:**
- 수치로 가치 증명 (시간 절약, 속도 향상 등)
- 유사 기술과의 비교
- 대상 독자별 활용 시나리오

**시작하기 섹션:**
- 복사-붙여넣기 가능한 설치 명령어
- Apple Silicon / Intel 등 환경별 분기 명시
- 설치 확인 명령어 + 예상 출력 포함

**실무 예제 섹션:**
- 최소 3개의 완전한 시나리오
- 각 예제: 학습 목표 → 시나리오 → 단계별 구현 → 결과
- 모든 코드: 복사-붙여넣기로 즉시 실행 가능

**베스트 프랙티스 섹션:**
- Before/After 패턴으로 최적화 표현
- 보안 고려사항 포함

**트러블슈팅 섹션:**
- 실제로 자주 발생하는 오류 우선
- 증상 → 원인 → 해결 순서

### 5단계: 두 파일 생성

강의 폴더를 생성하고 두 파일을 작성합니다.

**폴더 경로:** `lectures/[4자리번호]-[슬러그]/`

- 순번: `lectures/` 디렉토리의 기존 폴더 수를 확인하여 자동 부여 (0001, 0002, ...)
- 슬러그: 강의 주제를 영문 또는 한글 소문자 + 하이픈으로 변환
- 예: `lectures/0001-터미널-생산성-설정/`, `lectures/0002-fastapi/`
- 폴더 생성 시 `assets/`와 `examples/` 하위 디렉토리도 함께 생성

#### 파일 1: `lecture-script.md` — 강의자용 스크립트

강의자가 화면에서 명령어를 실행하며 녹화하는 라이브 코딩 형식에 최적화된 낭독 스크립트입니다.

**형식 규칙:**

1. **문체**: ~합니다 격식체 엄수
   - `~볼게요` → `~보겠습니다`
   - `~이에요` / `~예요` → `~입니다`
   - `~거든요` → `~기 때문입니다`
   - `~잖아요` → `~습니다`
   - `~돼요` / `~있어요` / `~없어요` → 격식체 변환
   - `~주세요` → `~하시기 바랍니다` 또는 서술형

2. **청중 참여 문구 금지**: 아래 유형은 작성하지 않음
   - 청중에게 묻는 질문 (`"~하세요?"`, `"~계신가요?"`)
   - 동의 구하는 문구 (`"~죠?"`, `"~맞죠?"`)
   - 참여 유도 (`"따라해보세요"`, `"해보시겠어요?"`)
   - 질문형 전환 (`"여기서 질문 드리겠습니다"`)
   - 대신 서술형으로 변환: `"개발자는 하루에 터미널을 수십 번 사용합니다."`

3. **큐 마커**: 섹션 흐름을 제어하는 큐 마커를 적극 활용
   - `[인트로]` — 오프닝
   - `[📌 섹션 전환]` — 새 섹션 시작
   - `[💻 데모]` — 화면 실행 구간
   - `[⏸️ 잠깐]` — 주의/부연 설명
   - `[🎯 핵심 포인트]` — 강조 사항
   - `[📝 코드 설명]` — 코드 상세 설명

4. **라이브 코딩 실행 서술**: `[💻 데모]` 구간에서 코드블록 전후에 화면 동작 서술
   - 실행 전: `"화면에서 다음 명령어를 실행합니다."` / `"터미널에 다음을 입력합니다."`
   - 실행 후: `"실행하면 [예상 결과]를 확인할 수 있습니다."` / `"출력에서 [핵심 사항]을 볼 수 있습니다."`

5. **섹션별 시간 표시**: `[약 N분]` 형식으로 각 섹션 예상 시간 표시
   - 총 60~90분 분량 (유튜브 롱폼 기준)

6. **"왜 → 어떻게" 순서**: 명령어/개념 설명 전 반드시 필요성 먼저 서술

**구조 예시:**
```markdown
# {주제} 강의 스크립트

> 예상 총 강의 시간: 약 N분

---

## [인트로] {제목} [약 5분]

{오프닝 - 수치와 함께 가치 제시}

---

## [섹션 1] {제목} [약 N분]

[📌 섹션 전환]

{도입 서술}

[💻 데모]

화면에서 다음 명령어를 실행합니다.

```bash
{명령어}
```

실행하면 {예상 결과}를 확인할 수 있습니다.

[🎯 핵심 포인트]

{강조 내용}
```

#### 파일 2: `README.md` — 시청자용 실용 가이드

시청자가 강의를 보면서 따라하거나, 이후 참조할 수 있는 압축 레퍼런스입니다.

**형식 규칙:**

1. **빠른 시작 체크리스트** — 파일 상단에 전체 작업 흐름을 체크박스로 요약
2. **설치 명령어 모음** — 모든 명령어를 환경별(macOS/Linux 등)로 정리, 복사-붙여넣기 즉시 실행 가능
3. **완성형 설정 파일** — `~/.zshrc`, `~/.gitconfig` 등 전체 내용 포함 (부분 발췌 금지)
4. **빠른 참조 표** — 단축키, 별칭, 옵션 등을 표 형식으로 정리
5. **트러블슈팅 표** — 문제 | 원인 | 해결 3열 표 형식
6. **환경 확인 명령어** — 설치/설정 검증에 사용하는 명령어 모음
7. **유용한 링크 표** — 도구명 | URL 표 형식

**구조 예시:**
```markdown
# {주제} 실용 가이드
