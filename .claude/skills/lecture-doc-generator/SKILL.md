# 강의 문서 자동 생성기

이 스킬은 기술 주제에 대한 실무 중심 한국어 강의 문서를 자동으로 생성합니다. 공식 문서를 참고하여 개요, 설치, 사용법, 실무 예제까지 포함한 완전한 학습 자료를 만듭니다.

## 워크플로우

### 1단계: 기본 정보 수집

사용자로부터 다음 정보를 수집합니다:

**필수 정보:**
- **주제어**: 강의할 기술/도구/라이브러리 이름 (예: "FastAPI", "Docker", "React Hooks")
- **간단한 설명**: 해당 기술이 무엇인지 한 줄로 (예: "Python 웹 프레임워크", "컨테이너 플랫폼")

**선택 정보:**
- **공식 문서 URL**: 있다면 제공 (없으면 자동 검색)
- **강조할 내용**: 특히 다루고 싶은 기능이나 주제
- **대상 청중**: 초급/중급/고급 개발자
- **실무 시나리오**: 실제 프로젝트에서 어떻게 사용하는지

**정보 수집 방법:**

사용자가 충분한 정보를 제공한 경우:
- 바로 2단계로 진행

사용자가 간단하게 요청한 경우 (예: "FastAPI 강의 자료 만들어줘"):
- 주제어를 파악하고 나머지는 자동으로 처리
- 공식 문서 자동 검색
- 표준 구조로 문서 생성

### 2단계: 공식 문서 조사

**WebSearch 사용:**
```
"{주제어} official documentation"
"{주제어} getting started guide"
"{주제어} tutorial"
```

**WebFetch 사용:**
제공된 URL 또는 검색된 공식 문서 URL에서 다음 정보 추출:
- 핵심 개념 및 용어
- 설치 방법 및 요구사항
- 기본 사용법 및 API
- 베스트 프랙티스
- 실전 예제

**중요 원칙:**
- 공식 문서의 정확성을 최우선으로
- 최신 버전 정보 확인
- 한국어로 의역하되 기술 용어는 원문 유지

### 3단계: 문서 구조 생성

**기본 템플릿 구조:**

```markdown
# {주제어} 완벽 가이드

> {한 줄 설명}

## 목차
[자동 생성된 목차]

## 1. 개요
### 1.1 {주제어}란?
### 1.2 주요 특징
### 1.3 왜 사용하는가?
### 1.4 사용 사례

## 2. 시작하기
### 2.1 사전 요구사항
### 2.2 설치 방법
### 2.3 초기 설정
### 2.4 첫 번째 프로젝트

## 3. 핵심 개념
### 3.1 기본 개념
### 3.2 주요 기능
### 3.3 작동 원리

## 4. 사용법
### 4.1 기본 사용법
### 4.2 주요 기능 활용
### 4.3 고급 기능
### 4.4 설정 및 커스터마이징

## 5. 실무 예제
### 5.1 예제 1: [실무 시나리오]
### 5.2 예제 2: [실무 시나리오]
### 5.3 예제 3: [실무 시나리오]

## 6. 베스트 프랙티스
### 6.1 성능 최적화
### 6.2 보안 고려사항
### 6.3 테스트 전략
### 6.4 프로덕션 배포

## 7. 트러블슈팅
### 7.1 자주 발생하는 오류
### 7.2 디버깅 팁
### 7.3 성능 문제 해결

## 8. 추가 학습 자료
### 8.1 공식 문서
### 8.2 추천 튜토리얼
### 8.3 커뮤니티 리소스

## 9. 결론
### 9.1 핵심 요약
### 9.2 다음 단계
```

**구조 조정 원칙:**
- 주제에 따라 섹션 추가/제거/재배치
- 프로그래밍 라이브러리: API 레퍼런스 추가
- DevOps 도구: 아키텍처 및 배포 전략 강화
- 개발 도구: 단축키 및 워크플로우 최적화 추가

### 4단계: 섹션별 콘텐츠 작성

각 섹션에 대해 다음 원칙을 따릅니다:

**개요 섹션 (1):**
- 기술의 배경과 목적 설명
- 유사 기술과의 비교
- 학습 곡선 및 난이도
- ROI (투자 대비 효과)

**시작하기 섹션 (2):**
- 명확한 단계별 설치 가이드
- 각 OS별 차이점 (macOS, Linux, Windows)
- 필수 의존성 및 도구
- 설치 확인 방법
- "Hello World" 수준의 간단한 예제

**핵심 개념 섹션 (3):**
- 기술의 내부 작동 원리
- 핵심 용어 및 개념 정의
- 아키텍처 다이어그램 (필요시 Mermaid 사용)
- 주요 컴포넌트 설명

**사용법 섹션 (4):**
- 기본부터 고급까지 단계별 진행
- 실행 가능한 코드 예제 포함
- 각 기능의 용도와 사용 시기
- 옵션 및 파라미터 설명

**실무 예제 섹션 (5):**
- 최소 3개의 실전 시나리오
- 완전한 코드 (복사-붙여넣기 가능)
- 각 예제의 학습 목표 명시
- 단계별 설명 및 주석
- 실행 결과 포함

**베스트 프랙티스 섹션 (6):**
- 실무에서 검증된 패턴
- 피해야 할 안티패턴
- 성능 최적화 팁
- 보안 가이드라인

**트러블슈팅 섹션 (7):**
- 자주 발생하는 오류 목록
- 각 오류의 원인 및 해결 방법
- 디버깅 전략
- 유용한 디버깅 도구

### 5단계: 문서 작성 및 저장

**파일 생성:**

작업 디렉토리에 `[4자리 제로필자리순번].[강의주제].md` 형식으로 파일 생성:
- 순번은 기존 강의 문서 개수를 확인하여 자동으로 부여 (0001, 0002, 0003, ...)
- 예: `0001.FastAPI.md`, `0002.Docker.md`, `0003.React Hooks.md`
- 기존 파일이 없으면 `0001`부터 시작

**Write 도구 사용:**
- 전체 문서를 한 번에 작성
- Markdown 형식 준수
- 적절한 헤딩 레벨 사용

**문서 품질 체크리스트:**
- [ ] 모든 코드 예제가 실행 가능한가?
- [ ] 기술 용어의 한국어/영어 병기가 일관적인가?
- [ ] 각 섹션이 논리적으로 연결되는가?
- [ ] 초보자가 따라 할 수 있는가?
- [ ] 실무 경험이 반영되었는가?
- [ ] 최신 버전 정보가 포함되었는가?
- [ ] 링크가 정확한가?

### 6단계: 사용자 검토 및 개선

**초안 완성 후:**

사용자에게 다음을 물어봅니다:
1. "문서가 생성되었습니다: `{파일명}`. 검토해주세요."
2. "추가하거나 수정할 내용이 있나요?"
3. "특정 섹션을 더 상세히 설명할까요?"
4. "실무 예제를 더 추가할까요?"

**개선 작업:**
- Edit 도구로 수정 (str_replace 사용)
- 전체 문서 재작성 금지
- 변경 사항을 명확히 안내

**반복:**
- 사용자가 만족할 때까지 개선
- 각 피드백에 구체적으로 대응

## 작성 가이드라인

### 한국어 작성 원칙

**기술 용어:**
- 널리 알려진 용어는 영어 그대로 (API, CLI, Docker, React)
- 첫 등장 시 한국어 설명 병기: "컨테이너(Container)"
- 이후 영어 용어 사용

**문체:**
- 존댓말 사용 ("~합니다", "~입니다")
- 명확하고 직접적인 표현
- 능동태 선호
- 불필요한 수식어 제거

**코드 설명:**
- 코드 블록에 언어 명시: ```python, ```javascript, ```bash
- 주요 코드에는 한글 주석
- 실행 방법 명시
- 예상 출력 포함

### 실무 중심 작성

**30년 풀스택 개발자 관점:**
- 이론보다 실전 위주
- 흔한 실수와 해결법
- 성능 및 보안 고려사항
- 프로덕션 환경 팁
- 팀 협업 관점

**라이브 코딩 대비:**
- 단계별로 따라 할 수 있게
- 각 단계의 결과 확인 방법
- 디버깅 포인트 제시
- 확장 가능한 예제

### 예제 작성 원칙

**실행 가능한 코드:**
- 복사-붙여넣기로 바로 실행 가능
- 필요한 import/require 모두 포함
- 파일 구조 명시
- 의존성 설치 명령어 제공

**점진적 복잡도:**
- 예제 1: 최소 기능 (Hello World)
- 예제 2: 기본 기능 활용
- 예제 3: 실무 수준 애플리케이션

**각 예제 구조:**
```markdown
### 예제 N: [제목]

**학습 목표:**
- 목표 1
- 목표 2

**시나리오:**
[실무 상황 설명]

**구현:**

1. 프로젝트 설정
   ```bash
   # 명령어
   ```

2. 코드 작성
   ```language
   # 코드
   ```

3. 실행 및 확인
   ```bash
   # 실행 명령어
   ```

**결과:**
```
[예상 출력]
```

**설명:**
[코드 동작 원리 설명]

**확장 아이디어:**
- 추가 기능 1
- 추가 기능 2
```

## 팁

**효율적인 문서 생성:**
- WebSearch로 최신 정보 확인
- WebFetch로 공식 문서 정확성 보장
- 사용자 요청이 간단해도 완전한 문서 제공
- 분량: 10,000-20,000자 (유튜브 롱폼 강의 수준)

**품질 보장:**
- 공식 문서 내용 최우선
- 검증된 예제만 포함
- 최신 버전 기준
- 실무 경험 반영

**사용자 경험:**
- 빠른 초안 생성 (15분 이내)
- 명확한 진행 상황 안내
- 구체적인 개선 제안
- 유연한 커스터마이징
